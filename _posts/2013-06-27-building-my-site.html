Building My Personal Website

This summer I decided to scrap my old portfolio and build a new one from scratch. When I first started I wanted to make sure I accomplished a few things:

  • Increase Readability
  • Add blogging functionality
  • Make it responsive

I'll go through each of these challenges and also explain some things I learned along the way.

Readability

As can be seen below, my previouis site suffered from a readability problem since the paragraph text was too light and crowded.

The helvetica neue was also not working for the paragaph text, so I decided to go with a serif font for paragraph text and a sans-serif font for headers. In this case I decided to go with Minion Pro as my serif font and Freight as my sans-serif font both of which were served through Typekit. Blogging

It was important for me to include blogging functionality, but I wasn't excited about using a full blown blog platform such as Wordpress. Jekyll provided just the right amount of customization and really gave me full control over the implementation. It also had the added benefit of being able to host on Github.

I wanted to host my site on Github in order to avoid paying the $5.99 I currently pay to GoDaddy every month. Github's documentation is fairly simple, I would say almost too simple for someone to get started. It turns out you have to follow some pretty specific instructions that I'll go through here.

Create a new repository that must follow the naming scheme “username/username.github.io” which for me would be “ivanrodriguez/ivanrodriguez5.github.io” which also serves as the link to the site. You must then install Jekyll on your computer with the ruby command:

gem install jekyll

In order to get Jekyll to watch directories for changes and have the CSS update run the command:

jekyll serve -w

Sometimes I'll get an error and it says that localhost:4000 is already in use. This is due to the fact that sometimes I either forget to kill the server when I shut down the terminal or I tried to start the server in an incorrect directory. To remedy this search for the PID with the command:

lsof -i TCP:4000

and kill the process with:

kill -9 PID

and that should solve the problem.

Responsive

I wanted my site to be responsive and adhere to a grid system. After some initial research on fluid grids and responsive web app design I decided to build my site on a 12-column responsive grid. I used Unsemantic which is a responsive implementation of the 960 Grid System.
